# Selenium Playwright Migrator — standard migration mode

### Breaking change

- Removed the Waves/Wavefront execution mode, its runtime state machine, lifecycle commands, specialized agents, partition dashboards, and synthetic evidence recovery paths.
- Existing migration workspaces should be refreshed with `kit update --backup` so obsolete OpenCode commands and migration-kit scripts are removed.

### Changed

- `selenium-pw-migrator run` now starts one linear full-source pipeline: analyze → generate → verify → proposals.
- `selenium-pw-migrator verify-project` and `verify-ts-project` are available as direct commands and map to their real verification modes.
- The standard final gate requires concrete orchestration, generated report, and successful project-verification artifacts. Missing evidence is a failure and must never be reconstructed by hand.
- OpenCode integration now uses four roles only: orchestrator, executor, reviewer, and watchdog.
- Adapter configuration now fails closed on overlapping scopes by default, rejects duplicate mapping keys, parses complex exact method signatures with balanced delimiters, and reports suppression-only empty tests honestly.

### Retained optimizations

- Optional representative pilot calibration before the full run.
- Project-local memory and reviewed config-delta merging.
- Scope checks, artifact validation, no-progress protection, and at most one bounded highest-payoff repair before a complete rerun.

### Validation

- Added standard-mode contract tests and a full-source smoke script.
- Packaging and CI no longer require removed partition-runtime files.

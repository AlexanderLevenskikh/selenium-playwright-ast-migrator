# Adaptive wave sizing

Wave size follows demonstrated quality, not impatience.

- Increase the next wave only after an accepted wave with stable or improving readiness and no new systemic root category.
- Keep size unchanged when the wave passes but soft debt or uncertainty remains.
- Split or shrink when root blockers grow, assertion preservation drops, scope expands, or remediation payoff becomes too diffuse.
- Never materialize a later wave without a valid predecessor `wave-acceptance.json` receipt.
- A budget-exhausted draft is a valid honest stop, not permission to call the migration complete.

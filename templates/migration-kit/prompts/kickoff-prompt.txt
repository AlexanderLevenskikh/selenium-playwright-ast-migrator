PRIMARY GUARDED MIGRATION-KIT PROMPT.
This is the canonical installed prompt for guarded artifact-only Selenium → Playwright migration runs. Do not delegate to legacy root `.agent-loops` prompts.

Read first:
- `{{WORKSPACE}}/AGENT_CONTRACT.md`
- `{{WORKSPACE}}/README.md`
- `{{WORKSPACE}}/agent-state.md`
- `{{WORKSPACE}}/current-ticket.md`
- `{{WORKSPACE}}/state/final-gate.md`
- `{{WORKSPACE}}/state/continuation-contract.md`, if present
- `{{WORKSPACE}}/state/continuation-decision.json`, if present
- `{{WORKSPACE}}/state/stop-policy-checklist.md`
- all files under `{{WORKSPACE}}/state/`

Start a guarded Selenium → Playwright migration run.

Mode: migration-artifact

Hard artifact-only boundary:
- Allowed writes: `{{WORKSPACE}}/**` only.
- Forbidden writes: real target project, production POM project, real Playwright test project, `Web/**`, `**/*.csproj`, `nuget.config`, root-level generated files, and migrator repository source code.
- "Write POM" means generated POM proposal/scaffold under `{{WORKSPACE}}/runs/<run-id>/generated-pom/**`, `{{WORKSPACE}}/runs/<run-id>/target-shadow/**`, or `{{WORKSPACE}}/proposals/**`.
- If a real target/POM/project change is required, do not apply it. Write a proposal under `{{WORKSPACE}}/proposals/**` and stop with `BLOCKED_BY_FORBIDDEN_WRITE`.
- Before each major action, restate which `{{WORKSPACE}}/AGENT_CONTRACT.md` rule allows it.
- After every edit and before any final success claim, run `{{WORKSPACE}}/scripts/check-scope.ps1`. Any changed file outside `{{WORKSPACE}}/**` makes the run failed.

Scope:
- Source Selenium project: {{SOURCE}}
- Target/generated Playwright project: {{TARGET}}
- Migrator config/profile: {{CONFIG}}
- Compiled migrator tool, if compiled-tool-only mode: {{COMPILED_TOOL}}
- Existing Playwright POM examples: {{TARGET_POM_EXAMPLES}}
- Workspace: {{WORKSPACE}}
- First output directory: {{OUTPUT}}

Expected first commands:
1. Run doctor/config validation for the scope.
2. Run the smallest useful migrate/verify/orchestrate command.
3. Build a migration board / explain TODO report if available.
4. Choose the next root-cause ticket from evidence, not from memory.
5. Update `{{WORKSPACE}}/agent-state.md`, `{{WORKSPACE}}/current-ticket.md`, `{{WORKSPACE}}/state/run-ledger.md`, `{{WORKSPACE}}/state/decision-log.md`, `{{WORKSPACE}}/state/handoff.md`, and `{{WORKSPACE}}/state/stop-policy-checklist.md`.

Rules:
- If status is `CONTINUE_AUTONOMOUSLY`, continue without asking the user whether to continue.
- If the batch is `NOT FINAL - INVESTIGATION RESULT ONLY` or `NOT RUNTIME READY`, keep going when the evidence names an allowed next action such as adapter-config mapping, target-shadow/POM scaffold proposal, discover-target, explain-todo, or another verification pass under `{{WORKSPACE}}/**`.
- Do not stop after a first-pass migrate/verify failure when the next root-cause ticket can be generated and executed from allowed source/config evidence.
- Do not stop at compile-green if the board still has actionable migration quality work.
- Do not edit migrator source code in migration-artifact mode.
- If compiled migrator tool is provided, run that tool directly and do not search for `Migrator.Cli`, `.sln`, `.csproj`, or migrator source code.
- Do not add or broaden suppressions to reduce TODO count.
- Never suppress FluentAssertions, NUnit assertions, assertion-like helpers, or business checks (`*.Should*`, `Should()`, `Assert*`, `Expect*`, `Equal*`, `Be*`, `Contain*`, validation/check helpers) unless a human explicitly approved that exact suppression with source evidence.
- TODO removed via suppression does not count as progress.
- TODO removed by weakening assertions, deleting actions, emptying tests, adding dummy target-known identifiers, or editing the real target project is a failed migration.
- `0 TODO` is valid only when config-validate/quality gates show no dangerous suppression, no empty meaningful tests, no suppressed side-effect dependency, no forbidden writes, and generated tests still contain source-backed actions/assertions.
- Prefer config fixes for project-specific selectors/helper mappings.
- Use engine fixes only in migrator-code mode for generic migrator limitations and add regression tests.
- Before stopping or handing off, update `{{WORKSPACE}}/state/stop-policy-checklist.md`.
- A handoff/report is not a stop reason by itself. Stopping requires a valid checked reason from `{{WORKSPACE}}/state/stop-policy-checklist.md`, max iterations, or a next action that is outside the allowed lane.
- After a non-final final-gate run, read `{{WORKSPACE}}/state/continuation-decision.json`. If it says `CONTINUE_REQUIRED`, execute exactly one next bounded action before any user-facing handoff. A response that only repeats `NOT FINAL` / `NOT RUNTIME READY` while continuation is required is a protocol violation.
- Do not claim FINAL unless `{{WORKSPACE}}/scripts/check-final-gate.ps1 -RequireOpenCodeExport -RequireExplainTodo -RequireVerificationArtifacts` passes. If strict evidence is not available, report `NOT FINAL - INVESTIGATION RESULT ONLY`.
- If interrupted, the next agent must be able to resume from workspace files.

# Migration PR Pack

`pr pack` turns migration run artifacts into a practical PR/review bundle. It is meant for the moment after a migration increment has produced reports, generated files, selector evidence, runtime feedback, and optionally an evidence zip.

```bash
selenium-pw-migrator pr pack \
  --input migration/runs/run-042 \
  --config ./adapter-config.json \
  --out migration/pr-pack \
  --format both
```

Mode-compatible form:

```bash
selenium-pw-migrator --mode pr-pack \
  --input migration/runs/latest \
  --config ./adapter-config.json \
  --out pr-pack \
  --format both
```

## Outputs

- `pr-summary.md` — human review overview with before/after metrics, changed/generated files list, risk summary, evidence pack link/manifest references, and suggested PR description.
- `pr-pack.json` — machine-readable PR pack for agents, CI, dashboards, or issue automation.
- `reviewer-checklist.md` — checklist for the reviewer.
- `suggested-pr-description.md` — copyable PR body.

## Inputs used when present

The command is read-only and scans the input directory for known migration artifacts:

- `runbook.md/json`
- `report-triage-decisions.md/json`
- `runtime-feedback-loop.md/json`
- `selector-evidence.md/json`
- `evidence-manifest.md/json`
- `verify-project-report.md/json`
- `project-verify-report.md/json`
- `report.md/json/txt`
- generated Migrator source files containing `Generated by Migrator` or `MIGRATOR:` markers

## Safety model

`pr pack` does not edit source tests, generated files, config layers, or product code. It only writes review artifacts under `--out`.

The bundle intentionally calls out missing evidence instead of pretending the migration is proven. For example, if selector evidence is missing, the PR pack adds a `missing-selector-evidence` risk and asks the author to run `selector evidence` before asking reviewers to accept locator mappings.

## Recommended production flow

```bash
selenium-pw-migrator runbook --input ./OldTests --out migration/runbook --format both
selenium-pw-migrator --mode migrate --input ./OldTests --config ./adapter-config.json --out migration/runs/run-042 --format both
selenium-pw-migrator selector evidence --input migration/runs/run-042 --config ./adapter-config.json --out migration/runs/run-042/selector-evidence --format both
selenium-pw-migrator report serve --input migration/runs/run-042 --static-only --out migration/runs/run-042/dashboard --format both
selenium-pw-migrator evidence pack --input migration/runs/run-042 --out evidence/run-042.zip
selenium-pw-migrator pr pack --input migration/runs/run-042 --config ./adapter-config.json --out migration/pr-pack --format both
```

Attach or link `pr-summary.md`, `suggested-pr-description.md`, and the evidence pack manifest in the PR.

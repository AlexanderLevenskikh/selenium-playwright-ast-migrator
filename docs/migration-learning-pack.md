# Migration Memory / Learning Pack

`learn pack` extracts reusable migration knowledge from a completed migration run. It turns proven selectors, helper semantics, POM patterns, and successful config knowledge into a reviewable learning pack.

```bash
selenium-pw-migrator learn pack \
  --input migration/runs/latest \
  --config ./adapter-config.json \
  --out learn-pack \
  --format both
```

Mode-compatible form:

```bash
selenium-pw-migrator --mode learn-pack \
  --input migration/runs/latest \
  --config ./adapter-config.json \
  --out learn-pack \
  --format both
```

## Inputs

The command looks for reusable knowledge in:

- `selector-evidence.md/json`
- `helper-inventory.md/json`
- `pom-index.generated.md/json`
- `config-proposals.md/json`
- `runtime-feedback-loop.md/json`
- `verify` / `project-verify` reports
- generated files marked with `Generated by Migrator`
- the current adapter config passed through `--config`

## Outputs

- `learn-pack.md` — human-readable summary of learned mappings, helper semantics, POM patterns, and reusable candidates.
- `learn-pack.json` — machine-readable learning pack.
- `learn-changelog.md` — changelog of what the Migrator learned from this run.
- `learning-safety-report.md` — redaction/safety review notes.
- `reusable-profile-layer.json` — candidate profile layer to review, diff, validate, and optionally install.

## Safety model

`learn pack` is read-only. It never edits source tests, generated tests, or adapter config files. It does not invent selectors. It intentionally does not export source-only identifiers (`SourceOnlyIdentifiers`), `SuppressedMethods`, or `SuppressedMethodPatterns` into the reusable profile layer because those are project-specific and can hide assertions if reused blindly.

Review loop:

```bash
selenium-pw-migrator learn pack --input migration/runs/latest --config ./adapter-config.json --out learn-pack --format both
selenium-pw-migrator --mode config-diff --before ./adapter-config.json --after learn-pack/reusable-profile-layer.json --out learn-pack/config-diff --format both
selenium-pw-migrator --mode config-validate --config learn-pack/reusable-profile-layer.json --validation-mode production --out learn-pack/config-validate --format both
```

Treat `reusable-profile-layer.json` as a candidate layer, not hidden behavior. Promote it to a shared profile only after `selector-evidence`, `helper-inventory`, `config-diff`, and `config-validate` support the change.

# Живой дашборд прогресса миграции

Это reference-документ, а не второй launch procedure. Канонический guarded launch остаётся в `docs/guarded-opencode-desktop-runbook.ru.md`.

## Зачем он нужен

Миграция может делать полезную работу и при этом выглядеть «застрявшей». Например, черновик уже создан для множества тестов, но текущая волна остановлена на улучшении маппингов или исправлении evidence. Дашборд показывает эти факты отдельно, чтобы пользователю не приходилось разбираться в `CONTINUE_REQUIRED`, JSONL и кодах quality budget.

Первый экран отвечает на четыре вопроса:

1. Где сейчас находится миграция?
2. Что уже сгенерировано?
3. Какая часть прошла защищённый процесс и принята?
4. Что harness делает дальше и нужно ли вмешательство человека?

У каждого важного показателя и этапа есть подсказка `?`. Отдельный индикатор прямо говорит, может ли harness продолжить сам или действительно нужно решение пользователя. Технические подробности остаются ниже в раскрывающихся секциях.

## Как считаются проценты

Дашборд специально показывает два разных прогресса:

- **Черновое покрытие** — для скольких тестов уже существует Playwright-output. Полезный результат виден даже во время дочистки.
- **Принятый прогресс** — тесты в волнах, которые прошли ограниченные quality и safety checks.

Большой общий процент — ориентировочная оценка процесса с большим весом принятого результата. Это навигация, а не заявление о production readiness.

## Этапы миграции

Интерактивная схема объясняет весь цикл:

1. Разделить тесты на небольшие волны.
2. Создать ограниченный черновик.
3. Улучшить маппинги, TODO, fallback и целостность scope.
4. Проверить качество, безопасность и runtime evidence.
5. Принять волну и перейти к следующей.

Текущий этап подсвечивается. Жёлтый quality-статус обычно означает «полезный черновик уже есть, но его надо дочистить до масштабирования», а не «миграция провалилась».

## Создать обычный снимок

```powershell
.\migration\scripts\build-harness-dashboard.ps1 -Workspace migration -Out dashboard/harness -Language ru
```

## Запустить постоянное обновление

Оставь команду работающей в отдельном терминале:

```powershell
.\migration\scripts\build-harness-dashboard.ps1 `
  -Workspace migration `
  -Out dashboard/harness `
  -Language ru `
  -Watch `
  -RefreshSeconds 5
```

Открой `migration/dashboard/harness/index.html`. Watcher пересобирает HTML/JSON/Markdown, а страница сама перечитывает локальный файл. Сервер не нужен. Остановка — `Ctrl+C`.

Не запускай `-Watch` внутри автономной agent-команды, которая должна завершиться: live-режим предназначен для отдельного пользовательского терминала. `/dashboard-harness` создаёт неблокирующий snapshot и печатает команду для live-режима.

## Что выводится

```text
migration/dashboard/
  i18n/
    en.json
    ru.json
  harness/
    index.html
    harness-dashboard.json
    harness-dashboard.md
```

Страница показывает короткие read-only превью максимум трёх сгенерированных `.cs`-файлов текущей волны. Пока идёт улучшение качества, в превью могут оставаться TODO.

## Правило локализации

English остаётся каноническим UI-языком, русский — полноценной локализацией. Сгенерированная страница предоставляет control `languageSelect` для мгновенного переключения. Машиночитаемые поля `phase`, `action`, `status` и имена gates не переводятся и остаются стабильными.

## Критерии готовности

- `index.html` открывается напрямую с диска;
- переключатель языка меняет English/Русский, включая подсказки;
- live-режим пересобирает файлы и пишет `HARNESS_DASHBOARD_REFRESHED`;
- черновой и принятый прогресс разделены;
- есть подсказки по этапам и превью сгенерированных тестов;
- `harness-dashboard.json` остаётся language-neutral;
- dogfood smoke пишет `evidence/harness-dashboard-smoke.md`.

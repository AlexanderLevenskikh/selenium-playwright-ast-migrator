# Report Serve Dashboard

`report serve` is the local dashboard entry point for reviewing migration run artifacts without sending project data anywhere.

```text
selenium-pw-migrator report serve --input migration/runs/latest --port 5077
```

Mode-compatible form:

```text
selenium-pw-migrator --mode report-serve --input migration/runs/latest --out report-dashboard --format both
```

Use `--static-only` or `--port 0` to generate files without starting a long-running local server:

```text
selenium-pw-migrator report serve --input migration/runs/latest --static-only --out report-dashboard
```

## What it creates

The command always writes a static dashboard first:

- `report-dashboard.html` — navigable local dashboard.
- `report-dashboard.md` — review-friendly text fallback when `--format text|both`.
- `report-dashboard.json` — machine-readable dashboard data when `--format json|both`.
- `report-dashboard-evidence.zip` — shareable evidence pack with dashboard files, run reports, generated migration artifacts, manifest, and checksums.

The server is optional. When `--port` is greater than zero, the CLI serves the output directory from `localhost` only. No external network dependency or upload is used.

## Dashboard sections

- Overview: files, tests, generated files, syntax/compile errors, TODO, unsupported actions, and unmapped targets.
- Quality trend: compares sibling run directories under the same parent folder.
- TODO explorer: groups `MIGRATOR:*` codes found in generated files and report artifacts.
- Unsupported actions: frequency, examples, and source locations.
- Unmapped targets: source expressions that still need profile/POM evidence.
- Verify/project-verify diagnostics: status and diagnostic counts.
- Runtime failures: groups from `runtime-failure-report.json` when available.
- Proposed next tickets: recommendations from migration-board/explain-todo/smoke-plan artifacts.
- Evidence pack: downloadable zip for PRs/issues.

## Safety model

- The command reads migration artifacts and generated files only.
- It does not edit source tests, config files, or target project files.
- It does not send data outside the machine.
- The evidence zip uses relative entry names and includes a manifest with SHA-256 checksums.
- Source repository files are not included unless they are generated migration artifacts marked by `Generated by Migrator` or `MIGRATOR:`.

## Missing artifacts

The dashboard degrades gracefully. For example, if runtime classification has not been run yet, the runtime section explains that `runtime-failure-report.json` is missing instead of failing the command.

# Selenium Playwright Migrator 0.0.0-preview.8

This preview promotes the npm wrapper from a local packaging experiment to a real public npm distribution channel, while keeping the same standalone no-.NET runtime payload underneath.

## Added

- Public npm package: `selenium-pw-migrator`.
- Preview install channel:

```bash
npm install -g selenium-pw-migrator@preview
selenium-pw-migrator --version
```

- GitHub Release `.tgz` asset for pinned npm-wrapper installs.
- Corporate Nexus/npm-proxy install support with npm config keys such as `selenium-pw-migrator-base-url`.
- Token-first npm publish workflow and optional Trusted Publishing/provenance mode.

## Fixed

- Preview npm versions are published with the `preview` dist-tag instead of `latest`.
- The publish workflow and local publish scripts reject prerelease versions when `latest` is selected accidentally.
- The tracked npm wrapper `bin/` sources are explicitly allowed despite the repository-wide `bin/` ignore rule.

## Corporate install example

```bash
npm install -g selenium-pw-migrator@preview \
  --registry=https://nexus.example/repository/npm-group/ \
  --selenium-pw-migrator-base-url=https://nexus.example/repository/migrator-releases/v0.0.0-preview.8
```

The npm registry/proxy serves the wrapper package. The `selenium-pw-migrator-base-url` setting points `postinstall` at the internal standalone archive mirror.

## Expected version output

```text
selenium-pw-migrator 0.0.0-preview.8+<commit>
distribution: standalone
runtime: win-x64
self-contained: true
publish-single-file: false
```

## Notes

- The npm package does not require the .NET SDK or .NET Runtime on the target machine.
- The native payload is still the standalone GitHub Release archive selected for `win-x64`, `linux-x64`, `osx-x64`, or `osx-arm64`.

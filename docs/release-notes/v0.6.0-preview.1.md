# Selenium Playwright Migrator 0.6.0-preview.1

This is the first public preview release of Selenium Playwright Migrator, a .NET 8 CLI toolkit for turning Selenium suites into measurable, reviewable Playwright migrations.

## Highlights

- Stable public path for **Selenium C# → Playwright .NET**, with NUnit as the default target framework and xUnit support called out in the framework matrix.
- Experimental preview paths for Playwright TypeScript and simple Java/Python Selenium fixtures.
- Reviewable adapter config format with explicit `SchemaVersion: adapter-config/v1`.
- Reports for unmapped targets, unsupported actions, generated TODOs, and migration quality.
- `--mode capabilities` for built-in source/target support matrices.
- Migration-kit templates for agent-assisted migration loops.
- Public packaging guardrails, local install smoke scripts, and bundle manifest/checksum generation.
- Demo repository assets and a screenshot-style walkthrough for first-time users.

## Try it

```bash
dotnet tool install --global SeleniumPlaywrightMigrator --version 0.6.0-preview.1
selenium-pw-migrator --help
selenium-pw-migrator --mode capabilities --out capabilities --format both
```

From source:

```bash
dotnet restore
dotnet run --project Migrator.Cli -- --help
```

## Recommended first workflow

```bash
selenium-pw-migrator --mode doctor --input ./SeleniumTests --config ./adapter-config.json --out doctor
selenium-pw-migrator --mode migrate --input ./SeleniumTests --config ./adapter-config.json --out pilot --format both
selenium-pw-migrator --mode verify --input ./SeleniumTests --config ./adapter-config.json --out pilot-verify --format both
```

Start with a small pilot folder. Inspect the reports, improve profile mappings using real selector/helper evidence, then regenerate.

## Known limitations

- The tool does not invent selectors.
- Best results require project-specific adapter/profile mappings.
- Generated TODOs are intentional review points when source truth is missing.
- Java, Python, and Playwright TypeScript support are preview/experimental paths.
- Some CLI implementation still lives in legacy `Program.cs` blocks and will be extracted incrementally.

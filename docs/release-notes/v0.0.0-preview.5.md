# Selenium Playwright Migrator 0.0.0-preview.5

This preview focuses on making Selenium Playwright Migrator easier to install and diagnose outside of .NET SDK-centric environments.

## Added

- Standalone self-contained release archives for Windows, Linux, and macOS.
- Windows and Unix standalone installers: `install-standalone.ps1` and `install-standalone.sh`.
- GitHub Release assets for versioned and alias standalone archives.
- `checksums.sha256` and `standalone-release-manifest.json` for release verification and internal mirrors.
- Rich `selenium-pw-migrator --version` diagnostics with commit, build time, distribution, runtime, self-contained mode, and `PublishSingleFile` state.
- Generic `-BaseUrl` / `--base-url` installer support for GitHub Releases, Nexus, or any flat static release directory.

## Fixed

- GitHub Releases now attach all standalone `.zip` / `.tar.gz` archives instead of only publishing installer scripts and the NuGet package.
- Windows standalone installer adds the user-local binary directory to `PATH` by default and updates the current PowerShell session.
- Local archive install smoke mode works through `-ArchivePath` / `--archive-path` with checksum verification.
- Release workflow verifies a flat `artifacts/github-release` staging directory before uploading release assets.

## Install

### Windows standalone

```powershell
$version = "0.0.0-preview.5"
$baseUrl = "https://github.com/AlexanderLevenskikh/selenium-playwright-ast-migrator/releases/download/v$version"
$installer = Join-Path $env:TEMP "install-standalone.ps1"

Invoke-WebRequest "$baseUrl/install-standalone.ps1" -OutFile $installer
& $installer -Version $version -BaseUrl $baseUrl

selenium-pw-migrator --version
```

### .NET tool

```powershell
dotnet tool update --global SeleniumPlaywrightMigrator `
  --source https://api.nuget.org/v3/index.json `
  --prerelease
```


Expected standalone version output includes:

```text
selenium-pw-migrator 0.0.0-preview.5+<commit>
distribution: standalone
runtime: win-x64
self-contained: true
publish-single-file: false
```

## Notes

- Standalone distribution does not require the .NET SDK or .NET Runtime on the target machine.
- The standalone bundle intentionally keeps `PublishSingleFile=false` because the Roslyn-based CLI needs real adjacent assemblies and bundled resources.
- Use `selenium-pw-migrator --version` in bug reports so the installed distribution and runtime are visible.

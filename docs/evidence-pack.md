# Evidence pack workflow

`evidence pack` creates a shareable zip from a migration run directory. It is meant for PRs, issues, handoffs, and agent/debug loops where reviewers need the same evidence without receiving the full source repository.

```bash
selenium-pw-migrator evidence pack \
  --input migration/runs/run-042 \
  --out evidence/run-042.zip
```

Mode-compatible form:

```bash
selenium-pw-migrator --mode evidence-pack \
  --input migration/runs/run-042 \
  --out evidence/run-042.zip
```

## What goes into the zip

The pack includes migration artifacts that are safe and useful for review:

- reports: `report.json`, verify/project-verify summaries, config-diff/doctor/runtime/report-serve outputs;
- generated Migrator output files when they are marked with `Generated by Migrator` or `MIGRATOR:*` comments;
- config/profile layers and dashboard/static report files;
- selected logs and runtime evidence such as trace zips, screenshots, videos, HAR files, and classification reports when they are already present in the artifact directory;
- `evidence-manifest.json`;
- `evidence-manifest.md`;
- `checksums.sha256`.

The command does not modify the source project or artifact directory.

## Source-file safety

By default, source-like files (`.cs`, `.ts`, `.tsx`, `.js`, `.java`, `.py`) are excluded unless they look generated by Migrator. This prevents accidentally attaching the original Selenium repository or product source files.

To intentionally include source-like files, pass:

```bash
selenium-pw-migrator evidence pack \
  --input migration/runs/run-042 \
  --out evidence/run-042-with-source.zip \
  --include-source
```

Review packs created with `--include-source` before external sharing.

## Redaction model

Text artifacts are copied into the zip after best-effort redaction:

- user home paths become `<USER_HOME>`;
- the artifact root path becomes `<ARTIFACT_ROOT>`;
- manifest paths are redacted with the same path redaction helper used by public reports.

Binary artifacts are included as-is and marked `binary-not-redacted` in the manifest. This includes Playwright traces, screenshots, videos, and HAR files. These files can contain URLs, screenshots, network metadata, and visible application state, so review them before sharing outside the team.

## Manifest and checksums

Every pack contains:

- `evidence-manifest.json` — machine-readable entry list, redaction metadata, SHA-256 checksums, and sensitive-content hints;
- `evidence-manifest.md` — reviewer-friendly summary;
- `checksums.sha256` — checksum list for packed entries and manifest files.

These files make the pack safe to attach to issues/PRs and easy to audit later.

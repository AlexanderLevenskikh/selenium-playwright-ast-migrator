# Security Policy

## Supported versions

Security fixes are handled for the latest preview release and the current `main` branch. Older preview builds may be superseded instead of patched in place.

## Reporting a vulnerability

Please report suspected vulnerabilities privately through your repository host's private vulnerability reporting flow or another trusted private channel maintained by the project owners. Do not publish exploit details in public issues before maintainers have had time to investigate.

Helpful reports include:

- affected version or commit;
- operating system and .NET SDK version;
- minimal reproduction steps;
- whether generated output, logs, reports, or packaged assets expose sensitive data.

## Handling sensitive migration data

Migration inputs may contain private selectors, URLs, screenshots, logs, and project-specific source code. Before sharing reports publicly, review generated artifacts and redact project-specific paths, hosts, credentials, and customer data.
